# Subgroup Fairness Analysis (Identity Hate)

This notebook evaluates **fairness disparities** across identity subgroups for the `identity_hate` label using model predictions generated in earlier sections (TF-IDF and Transformer pipelines).

The goal is to quantify whether models behave differently across identity-related groups, with a focus on **error asymmetry** rather than just overall performance.

---

## What This Section Does

This notebook performs a structured subgroup analysis by:

* Filtering the dataset by identity indicators (e.g., mentions of specific groups)
* Computing confusion-matrix-based metrics within each subgroup
* Comparing **False Positive Rate (FPR)** and **False Negative Rate (FNR)** across models
* Exporting results for reporting and visualization

This directly supports the project’s fairness objective of identifying **systematic bias in toxicity detection**, particularly for identity-related language fileciteturn0file1.

## Fairness Metrics

We compute subgroup-level confusion matrix statistics:

* True Positives (TP)
* False Positives (FP)
* True Negatives (TN)
* False Negatives (FN)

From these, we derive:

* **False Positive Rate (FPR)** = FP / (FP + TN)
* **False Negative Rate (FNR)** = FN / (FN + TP)

### Why These Metrics

* **FPR** captures *over-flagging*: non-toxic comments incorrectly labeled as toxic
* **FNR** captures *under-flagging*: toxic comments that the model misses

In fairness analysis:

* High FPR for a subgroup → potential *over-censorship*
* High FNR for a subgroup → potential *under-protection*

---

## Workflow

### 1. Subgroup Filtering

For each identity group:

* Select rows where that identity indicator = 1
* Skip groups with no representation

### 2. Metric Computation

For each model and subgroup:

* Compute confusion matrix
* Extract TP, FP, TN, FN
* Calculate FPR and FNR

### 3. Aggregation

Results are stored in a structured DataFrame:

* `model`
* `group`
* `TP`, `FP`, `TN`, `FN`
* `FPR`, `FNR`

Converted to percentage form for readability.

### 4. Output Tables

Two pivot tables are created:

* FPR by model × subgroup
* FNR by model × subgroup

Saved as:

* `fpr_output.xlsx`
* `fnr_output.xlsx`

## Interpretation Guidelines

When reviewing outputs:

* Compare **relative differences** across groups rather than absolute values alone
* Look for consistent patterns (e.g., one model systematically higher FPR)
* Identify outlier subgroups with unusually high error rates

Typical findings in this domain include:

* Higher FPR on identity-related text (false flagging)
* Tradeoffs between FPR and FNR across model families
