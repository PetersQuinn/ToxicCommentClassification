# Outputs

We keep this folder focused on the early data workflow for the larger project.

Tracked outputs:
- `01_data_audit/` holds schema checks and quick dataset summaries
- `02_eda/` holds lightweight EDA tables
- `03_cleaned_data/README.md` explains the cleaned parquet exports we generate locally
- `04_option_a_tabular/` keeps small metadata and diagnostics for the tabular branch
- `07_figures/eda/` keeps the retained EDA figures

Heavy parquet files, sparse matrices, vectorizers, and other bulk artifacts stay local and are ignored for GitHub.
