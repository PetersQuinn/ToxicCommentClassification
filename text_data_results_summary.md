## 4. TF-IDF results summary
The main TF-IDF task appears to be the six-label multilabel problem, with binary models used as secondary follow-ups. The retained classical families were logistic regression, linear SVM, and complement Naive Bayes. On the final labeled test split, the latest completed step-6 run shows logistic regression as the best TF-IDF family with `macro_pr_auc = 0.5758156051899487` and `macro_f1 = 0.5412`, ahead of linear SVM at `0.5629678012328626` and `0.5164`, and well ahead of complement NB at `0.45610982977195347` and `0.4326`. There are two completed step-6 logs (`125413` and `130116`) with matching metrics, so the later `130116` run appears to be the most final duplicate rather than a conflicting result.

The validation artifacts show the same ordering on AP. Across tuned cross-validation, logistic regression had mean `macro_pr_auc = 0.640051301668601`, ahead of linear SVM at `0.622398316882773` and complement NB at `0.52288071554554`. Linear SVM was still competitive because it slightly led logistic regression on validation `macro_f1` (`0.600155338902361` versus `0.590736359143268`) and subset accuracy, so the repo evidence supports saying logistic regression remained competitive and ultimately preferable when AP is the primary decision metric, while linear SVM is a reasonable runner-up if thresholded classification behavior matters more. Complement NB looks more like a recall-heavy but low-precision fallback than a final contender.

Threshold tuning materially affected interpretation. The TF-IDF pipeline explicitly tuned thresholds after AP-based model selection, and the selected logistic-regression thresholds were fairly high for all labels: toxic `0.67`, insult `0.70`, obscene `0.65`, identity_hate `0.73`, severe_toxic `0.83`, and threat `0.83`. At those validation-selected thresholds, logistic regression achieved validation toxic `precision = 0.7896`, `recall = 0.7443`, `f1 = 0.7663`; severe_toxic `precision = 0.4234`, `recall = 0.6166`, `f1 = 0.5020`; and identity_hate `precision = 0.3985`, `recall = 0.5371`, `f1 = 0.4575`. Runtime is a major TF-IDF advantage: the final export-only test pass took about `49s` for logistic-regression multilabel fit plus evaluation, `53s` for linear SVM, and `10s` for complement NB.

## 5. Transformer results summary
The main transformer result is still the shared six-label DistilBERT model (`distilbert-base-uncased`) trained with 3-fold cross-validation and 3 epochs per fold, selected by validation mean average precision. The comparability fix did not change model selection because AP is threshold-free: the shared model remains at `mean_average_precision = 0.680587031044503` on the labeled test split, with `mean_roc_auc = 0.9852625356888168`. What changed is the operating point. Using validation-only threshold tuning that mirrors the TF-IDF rule, the shared model now uses per-label thresholds of toxic `0.50`, severe_toxic `0.25`, obscene `0.41`, threat `0.30`, insult `0.38`, and identity_hate `0.31`, yielding `macro_f1_at_operating_threshold = 0.611042367088549` instead of `0.6068714137600176` at the original fixed `0.5`.

The biggest threshold shifts were on the rarer labels, which moved toward higher recall. On the held-out test split, shared severe_toxic recall rose from `0.3243` to `0.7248` and F1 rose from `0.3707` to `0.4189`; shared identity_hate recall rose from `0.4888` to `0.6657` and F1 rose from `0.5928` to `0.6320`. Not every label improved on test F1 after tuning: threat dropped from `0.5791` to `0.5331`, and obscene and insult each lost a small amount of precision-heavy F1 despite higher recall. That pattern is worth stating explicitly in the paper: validation-based threshold tuning made the transformer comparisons scientifically fairer relative to TF-IDF, but it did not uniformly improve every held-out thresholded metric.

The binary transformer runs remain better interpreted as specialization studies than as the main recommendation. Each binary model still used only 1 epoch and trained in roughly `174` to `175` seconds, while the shared multilabel run took `1642.632747648` seconds total (`27.377212460800003` minutes) and peaked at about `2781 MB` GPU memory. After threshold tuning, the binary suite still averaged only `0.6440112268508875` AP across labels, below the shared model's `0.680587031044503`, and it averaged `0.5911999526448233` F1 at the operating threshold. The shared model now beats the binary model on both AP and tuned F1 for severe_toxic, obscene, threat, and insult; binary `identity_hate` keeps a tiny F1 edge (`0.6344` versus `0.6320`) but still loses on AP, and `binary_toxic` could not be re-tuned because `validation_predictions.csv` was not present in the tracked repo.

## 6. Comparison and recommendation-relevant takeaways
- Best raw performer: the shared DistilBERT multilabel model, with held-out test `mean_average_precision = 0.680587031044503`, beats the best TF-IDF model (logistic regression at `0.5758156051899487`) by about `0.1048` AP.
- Best classical baseline: TF-IDF logistic regression. It wins the TF-IDF family on final-test AP and also leads TF-IDF validation AP, while remaining much cheaper than the transformer models.
- Fastest / cheapest approach: complement NB is the absolute fastest TF-IDF model, but logistic regression is the better practical cheap option because it stays under a minute for final fit plus eval while materially outperforming complement NB.
- Most interpretable approach: TF-IDF logistic regression. It is a sparse linear model with coefficient-based interpretability and still the strongest classical baseline.
- Most practical one-model deployment candidate: shared DistilBERT, not the six binary transformers. It gives the best AP and avoids six-model training, monitoring, and maintenance.
- Most practical low-cost deployment candidate: TF-IDF logistic regression, especially if CPU-only serving, training speed, or transparency matters more than the last `~0.105` AP.
- Whether transformer gains justify cost: probably yes if the paper prioritizes predictive quality. The shared transformer improves materially over the best TF-IDF baseline and also beats the binary-transformer suite on AP despite being simpler to maintain than six binary models.
- Threshold comparability: largely fixed. TF-IDF and transformers are now both evaluated with post-training, validation-selected thresholds; the one remaining exception is `binary_toxic`, which stayed at `0.5` because its validation probability CSV is missing.
- Threshold tradeoff pattern: transformer tuning mostly lowered thresholds on rare labels, which raised recall and predicted positive rate. Shared severe_toxic moved from `0.5` to `0.25` and shared identity_hate from `0.5` to `0.31`, improving recall materially at the cost of precision.
- False-positive / false-negative implication: after tuning, shared `identity_hate` becomes less conservative (`precision = 0.6015`, `recall = 0.6657`) than the old fixed-`0.5` operating point, while binary `identity_hate` still offers a tiny additional F1 gain (`0.6344` vs `0.6320`) but lower AP and a slightly higher predicted positive rate.
- Secondary task note: the TF-IDF final exports concentrate on `identity_hate` predictions for downstream use, but those export artifacts do not overturn the main text-model recommendation because the shared transformer is still much stronger on `identity_hate` AP.

## 8. Suggested items for the main paper vs appendix

### Main paper
- Shared DistilBERT was the best overall text model on the held-out test set with `mean_average_precision = 0.680587031044503`.
- TF-IDF logistic regression was the best classical baseline with `macro_pr_auc = 0.5758156051899487`, outperforming linear SVM and complement NB.
- The shared transformer beat the six one-label binary transformers on AP for every label, so one shared model is easier to justify than maintaining six binary models.
- TF-IDF models were dramatically cheaper to fit and evaluate than the transformer models, with logistic regression finishing in under a minute versus roughly `27.4` minutes of GPU-backed training for the shared transformer.
- Threshold handling is now mostly aligned across families: TF-IDF and transformer results both use post-training, validation-selected thresholds, with the single caveat that `binary_toxic` remained at `0.5` because its validation prediction export is missing.
